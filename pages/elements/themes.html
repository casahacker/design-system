themes

Light e dark theme controlados por atributo data-theme no html. Tokens semânticos garantem que componentes nunca precisam saber qual tema está ativo.

temas ativos

01

Dois temas em produção, ambos cobrindo a paleta completa. Dark theme segue prefers-color-scheme como default.

// light · default

claro

background CSS, texto Dos, destaques Code.

// dark

escuro

background Dos, texto CSS, mesmo Code.

trocando de tema

02

Tema é controlado por atributo no <html>. Persistido em localStorage. Inicial respeita prefers-color-scheme.

/* CSS: tokens redeclarados */ html[data-theme="dark"] { --background: var(--ch-dos); --text-primary: var(--ch-css); /* etc */ } // JS document.documentElement.setAttribute("data-theme", "dark");

filosofia

03

Componentes não conhecem o tema. Eles consomem tokens semânticos (--text-primary, --layer-01) que mudam de valor conforme o atributo. Trocar de tema = trocar tokens, zero refactor de componente.

quando usar cada tema

04

use

  • Light por padrão em produtos de produtividade e leitura
  • Dark por padrão em IDEs, dashboards de monitoramento, conteúdo notural
  • Sempre permitir o usuário trocar
  • Respeitar prefers-color-scheme do SO

não use

  • Forçar um tema sem opção de troca
  • Cores fixas (cruas) que não respondem ao tema
  • Mensagens que pressupõem qual tema está ativo ("clica no botão escuro")

código

05
/* alternar tema via JS */ const cur = document.documentElement.getAttribute("data-theme"); if (cur === "dark") { document.documentElement.removeAttribute("data-theme"); localStorage.setItem("chds-theme", "light"); } else { document.documentElement.setAttribute("data-theme", "dark"); localStorage.setItem("chds-theme", "dark"); }
