como contribuir

01
  1. Fork do repo no github
  2. Cria branch a partir de main: git checkout -b feat/meu-componente
  3. Faz a mudança seguindo o component checklist
  4. Abre PR descrevendo o que mudou e por quê
  5. Aguarda review (em geral 3-5 dias úteis)

áreas que aceitam contribuição

02

componentes novos

Submeta proposta antes via issue. Precisa passar pelo PDLC.

melhorias de a11y

Sempre bem-vindas. Cite o critério WCAG no PR.

correções de bug

Pode ir direto pra PR. Inclua reproduction.

docs

Typos, exemplos, traduções, melhorias de conteúdo.

performance

Otimizações de CSS/JS desde que mantenham legibilidade.

submarcas/impressos

Novos templates, exemplos de uso, especificações.
