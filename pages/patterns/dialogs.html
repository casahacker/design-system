modal — quando

02

use

  • Confirmação destrutiva (deletar conta)
  • Formulário curto fora do contexto
  • Conteúdo crítico que precisa decisão

não use

  • Notificação simples — usa toast
  • Conteúdo longo — usa página
  • Pré-requisito de fluxo — usa wizard

toast — quando

03

use

  • Feedback rápido pós-ação ("salvo")
  • Notificação não-bloqueante
  • Confirmação leve

não use

  • Erro crítico — usa modal
  • Conteúdo persistente — usa notification inline
  • Empilhar muitos toasts simultaneamente
